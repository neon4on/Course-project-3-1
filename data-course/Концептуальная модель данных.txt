Концептуальная модель — это абстрактное представление системы, без технических деталей (типов данных, ключей и т.д.). Она описывает основные сущности и связи между ними.

Сущности и их описание
Пользователь (User)
Пользователь может регистрироваться в системе, совершать заказы, добавлять товары в корзину и получать подтверждения.

Категория (Category)
Категория объединяет продукты. Например, "Напитки", "Пицца".

Продукт (Product)
Продукт — это товар, который может быть добавлен в заказ или корзину. У каждого продукта есть название, изображение и принадлежность к категории.

Ингредиент (Ingredient)
Ингредиент — это компонент продукта (например, сыр, ветчина). Каждый ингредиент имеет цену и может быть добавлен в продукт.

Корзина (Cart)
Корзина содержит товары, добавленные пользователем. У каждой корзины есть токен и привязка к пользователю.

Товар в корзине (CartItem)
Элемент корзины содержит информацию о продукте, его количестве и дополнительных ингредиентах.

Заказ (Order)
Заказ включает данные пользователя (имя, адрес, телефон), товары и их количество. Также содержит статус заказа и сумму.

Код подтверждения (VerificationCode)
Код подтверждения связан с пользователем и используется для верификации.

История (Story)
История отображает содержимое (например, акции или новости) и состоит из элементов.

Связи между сущностями
Пользователь — Заказ (1:M)
Один пользователь может сделать несколько заказов.

Пользователь — Корзина (1:1)
У каждого пользователя есть одна корзина.

Категория — Продукт (1:M)
Одна категория может содержать несколько продуктов.

Продукт — Ингредиенты (M:N)
Один продукт может содержать несколько ингредиентов, и один ингредиент может быть частью нескольких продуктов.

Корзина — Товар в корзине (1:M)
В одной корзине может быть несколько товаров.

Заказ — Товары (JSON)
Товары в заказе представлены в виде списка.

История — Элементы истории (1:M)
История состоит из нескольких элементов.
